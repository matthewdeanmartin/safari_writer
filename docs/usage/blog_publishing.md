# Blog Publishing with Git

Safari Writer's **Git Publish** screen lets you commit and push blog posts to a
remote repository without leaving the app. This is designed for static-site
workflows such as **Pelican**, Jekyll, Hugo, or any setup where pushing to a git
remote triggers a site build.

## Quick workflow

1. Write your blog post and save it inside your blog's git repository.
1. Press **Ctrl+P** (or **P** from the Main Menu) to open the Print / Export menu.
1. Press **G** — "Git Publish (commit & push)".
1. The Git Publish screen opens with two panes:
   - **Status** — the current working tree (staged, modified, and untracked files).
   - **Recent Commits** — the last 20 commits in the repository.

### Step-by-step in the Git Publish screen

| Key | Action |
|-----|--------|
| **S** | Refresh the working tree status |
| **A** | Stage all changes (`git add -A`) |
| **C** | Open the commit message prompt and commit |
| **P** | Push to the remote (origin, current branch) |
| **U** | Pull the latest commits from the remote |
| **H** | Refresh the commit history pane |
| **R** | Change the repository path |
| **F1** | Show the full help screen |
| **Esc** | Return to the previous screen |

A typical session looks like this:

```
A  →  Stage everything
C  →  Type: "Add post: My new article"  →  Enter
P  →  Push — your post is live
```

## Repository auto-detection

When Git Publish opens it walks up the directory tree from the saved document's
location to find the nearest `.git` folder. If no document is saved yet, it
checks the current working directory. If no repository is found, press **R** to
enter the path manually.

## Pelican-specific notes

Pelican reads content from a `content/` directory. A typical repository layout:

```
my-blog/
  content/
    posts/
      my-article.md        ← write this in Safari Writer
  output/                  ← generated by Pelican
  pelicanconf.py
```

**Recommended workflow:**

1. Write your post in Safari Writer and save it as `content/posts/my-article.md`.
1. Export to Markdown first if you wrote in `.sfw` format (**M** in the Print menu),
   then save the `.md` file inside `content/`.
1. Open Git Publish (**G**), stage (**A**), commit (**C**), and push (**P**).
1. Your CI/CD pipeline (GitHub Actions, Netlify, etc.) picks up the push and
   rebuilds the site automatically.

> **Push = Publish.** Pressing **P** in Git Publish runs `git push` to the
> configured remote. If your remote triggers a build (e.g. GitHub Pages,
> Netlify, or your own server hook), the post goes live as soon as the push
> completes.

## Requirements

Git Publish requires the **GitPython** library:

```
uv add gitpython
```

If GitPython is not installed, the screen will show an error when you try to
use any git action.

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| "No git repository found" | Press **R** and enter the path to your blog repo |
| "No remote configured" | Add a remote: `git remote add origin <url>` |
| Push fails with auth error | Set up SSH keys or a credential helper for your remote |
| Commit fails with empty message | Type a non-empty message and press Enter |
